Control is sized incorrectly when container ScaleMode is not Twips

Summary

Id:1411.1
Type: Bug
Current Status:Open

Detail

19 Mar 2003OpenKim Pedersen

Just wanted to let you know I found a bug in the ComboBoxEx control. If you set the ScaleMode of the container of a ComboBoxEx control to fx. pixels the ComboBoxEx resizes itself to a pretty large control (in Height). I figure you hardcoded the height of the control into the resize of the UserControl as the API created ComboBoxEx control itself doesn't resize. Haven't looked in the source for it, but I figure you will easily work it out when you get the chance.
